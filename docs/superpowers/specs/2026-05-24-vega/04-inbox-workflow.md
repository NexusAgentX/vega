# 04 — Inbox 审核流水线 + 热更新

## 审核流水线

```
外部 agent / 用户导入
      │
      ▼
┌─────────────┐
│   INBOX     │  自动关联建议（规则 + LLM）
│ status: new │  "建议关联 doc-042, doc-107"
└──────┬──────┘
       │
       ▼  人工审阅（默认模式）
┌─────────────────────────────────┐
│  vega review                    │
│  待审: doc-089                  │
│  建议关联: doc-042, doc-107      │
│  [A] 通过  [R] 拒绝  [E] 编辑   │
└────────────┬────────────────────┘
             │
             ▼
┌─────────────┐          ┌──────────────┐
│   ACTIVE    │  AI对话   │  YOLO 模式   │
│  reviewed   │ ◀──────── │  跳过审核    │
│             │  动态更新  │  直接入库    │
└─────────────┘          └──────────────┘
```

## 两种入库模式

| 模式 | 流程 | 适用场景 |
|---|---|---|
| 默认（审核） | inbox → 人工 review → active | agent 自动抓取，人把关 |
| YOLO | 直接 active | 信任来源，或人手动导入 |

## 热更新机制

仅在 Core 本体 AI 对话中触发：

1. 用户与 AI 对话时，根据反馈修正 metadata（标签、标题等）
2. 建立/断开文档关系
3. 更新摘要

CLI 和外部 agent 调用**不触发**热更新。
